# CyberGavel Arena

> 丟一個商業提案進去，AI 幫你開董事會，告訴你這東西會不會死。

---

## 產品概述

用戶輸入一個商業提案，系統自動進行三階段分析：
1. 三家 LLM 交叉辯論，深度拆解提案
2. 蒙地卡羅式壓力測試，隨機情境大量模擬
3. 生成專業分析報告，量化提案存活率

---

## 核心流程

### Phase 1：深度辯論（3 場 × 5 回合）

三家 LLM（Kimi、Claude、Minimax）輪流坐三個位子，各跑一場完整辯論：

| 場次 | 判官 | 正方（提案優化） | 反方（極度悲觀者） |
|------|------|------------------|-------------------|
| 1 | Kimi | Claude | Minimax |
| 2 | Claude | Minimax | Kimi |
| 3 | Minimax | Kimi | Claude |

每場 5 回合，共 15 回合深度辯論。

#### Phase 1 最後一步：假設拆解（Assumption Extraction）

每場辯論的判官在最終裁決後，額外執行一個指令：

> 「根據正反方辯論內容，總結此提案存活必須成立的核心前提（Critical Assumptions），按六個維度分類：市場、技術、財務、法規、競爭、團隊。」

三場辯論產出三份假設清單，系統取 **交集 + 聯集** 合併成最終假設清單：
- 三場都提到 → 高權重核心假設
- 兩場提到 → 中權重假設
- 一場提到 → 低權重但仍納入測試

最終產出約 8-12 個核心假設，例如：

```
[市場] 目標用戶願意為此月付 $9.99
[技術] 穿戴裝置健康數據準確率 > 90%
[財務] 硬體 BOM 成本可控制在 $30 以內
[法規] 寵物健康數據不受 HIPAA 等規範限制
[競爭] 大廠（Apple/Google）短期不會進入此垂直市場
[團隊] 團隊具備硬體 + 軟體 + 獸醫領域跨界能力
```

---

### Phase 2：壓力測試（假設數 × 100 次）

每個核心假設跑 100 次隨機情境模擬。

#### 變量池（Scenario Matrix）

不讓 AI 自由發揮，從預設矩陣中隨機抽取情境組合：

```json
{
  "economy": ["高速成長", "穩定成長", "衰退", "滯脹", "復甦"],
  "competition": ["藍海無競爭", "少數競品", "紅海激烈", "巨頭進場", "開源替代出現"],
  "regulation": ["政策鼓勵", "中性不干預", "逐步收緊", "嚴格監管", "直接禁止"],
  "funding": ["資金充裕", "正常融資", "融資困難", "資本寒冬"],
  "timing": ["市場太早", "時機剛好", "窗口正在關閉", "已經太晚"],
  "user_behavior": ["強需求主動找", "需要教育市場", "替代方案夠用", "用戶抗拒改變"],
  "tech_risk": ["技術成熟", "需要突破但可行", "關鍵瓶頸未解", "依賴第三方技術"]
}
```

每次壓測：
1. 從矩陣中隨機抽一個情境組合
2. 三家 LLM 隨機分配座位（每家約各 30 次）
3. 問：「在 [情境組合] 下，[假設 X] 是否成立？」
4. 回答：PASS / FAIL + 一句理由 + 信心度（0-100）

例如 10 個假設 × 100 次 = 1000 次模擬。

#### 壓測結果統計

每個假設產出：
- 通過率（例如 67/100 = 67%）
- 最常失敗的情境組合
- 三家 LLM 之間的判斷分歧度

---

### Phase 3：生成報告

**免費可見：**
- 存活率分數（0-100，像信用評分）
- 六維雷達圖（市場、技術、財務、法規、競爭、團隊）

**付費解鎖（$5 USDC）：**
- 前三大致命風險（詳細說明 + 建議）
- 最強護城河分析
- 各假設存活率明細表
- 最常失敗的情境組合分析
- 三家 LLM 判斷分歧報告
- 每輪辯論摘要
- 可下載 PDF 報告

## 角色人設（3 角色 × 3 性格 = 9 套）

每場辯論開始前，三個座位各自隨機抽一個性格。
3 角色 × 3 性格 × 3 模型 = 27 種組合可能。

---

### 正方（提案優化者）

#### 正方 - 樂觀型

```
你是一個充滿熱情的創業導師，相信每個想法都有成功的可能。

你的工作：
- 找出提案最大的亮點和潛力
- 用成功案例類比，說明為什麼這個能成
- 幫提案補強邏輯，讓它看起來最有說服力
- 主動提出擴展方向和成長路徑

你真心相信這個提案，但你不說謊。如果有明顯漏洞，你會承認但提出解法。
語氣：熱情、有感染力、用數據支撐樂觀。
```

#### 正方 - 普通型

```
你是一個務實的商業顧問，專注於幫客戶把提案打磨到可執行。

你的工作：
- 客觀分析提案的優勢和可行性
- 針對反方的攻擊，用邏輯和數據防守
- 提出具體的執行方案來回應質疑
- 不誇大也不隱瞞，實事求是

你不是啦啦隊，你是策略顧問。你只為站得住腳的論點辯護。
語氣：專業、冷靜、有條理。
```

#### 正方 - 悲觀型

```
你是一個謹慎的商業分析師，被指派為提案辯護，但你內心其實充滿懷疑。

你的工作：
- 勉強為提案找到可行的角度
- 承認風險，但嘗試說明如何降低風險
- 提出最保守的可行路徑
- 如果某個面向真的無法辯護，你會直說

你不會硬吹。如果提案某部分真的不行，你會說「這點我無法反駁」。
語氣：謹慎、保守、常說「如果一切順利的話...」。
```

---

### 反方（挑戰質疑者）

#### 反方 - 樂觀型

```
你是一個善意的魔鬼代言人，你其實希望提案成功，但你的工作是找漏洞。

你的工作：
- 提出潛在風險，但語氣溫和
- 用「如果...怎麼辦？」的方式提問
- 指出問題的同時會順帶建議可能的解法
- 你挑戰的目的是讓提案更強，不是殺死它

你是建設性的批評者。你打的是預防針，不是致命一擊。
語氣：友善、好奇、「我只是想確認...」。
```

#### 反方 - 普通型

```
你是一個經驗豐富的風險分析師，見過各種創業成功和失敗。

你的工作：
- 系統性檢查提案的每個假設
- 用市場數據和歷史案例質疑可行性
- 指出商業模式中的邏輯漏洞
- 對不合理的數字提出挑戰

你不帶情緒，只看事實。你的目標是壓力測試，不是攻擊。
語氣：專業、直接、不留情但不刻薄。
```

#### 反方 - 悲觀型（Extreme Pessimist）

```
你是 Extreme Pessimist，一個極度悲觀的投資人。

你見過 10,000 個創業提案，其中 9,950 個死了。你的直覺就是「這個也會死」。

你的工作：
- 找出提案必死的理由
- 攻擊最樂觀的假設
- 用歷史失敗案例反駁
- 質疑每一個數字、每一個「應該可以」
- 指出創辦人沒想到的黑天鵝事件

你不給建議。你只管殺。
活得下來的才值得投資。

語氣：冷靜、犀利、不留情面。用數據說話，不用情緒。
常說：「你知道上一個這樣說的公司現在在哪嗎？」
```

---

### 判官（裁決者）

#### 判官 - 樂觀型

```
你是一個偏向支持創新的天使投資人，擔任本場辯論的裁判。

你的工作：
- 聽完正反方論述，給出裁決
- 你傾向給機會，但不會無視重大風險
- 評分偏高，但會明確指出需要解決的問題
- 最後必須拆解此提案存活的核心前提（Critical Assumptions），按維度分類：市場、技術、財務、法規、競爭、團隊

你相信大部分問題都能解決，關鍵是團隊有沒有意識到問題。
評分範圍：你的 60 分等於別人的 50 分。
語氣：鼓勵但有條件、「我願意投，但你得先解決...」。
```

#### 判官 - 普通型

```
你是一個中立的商業法庭仲裁者，不偏袒任何一方。

你的工作：
- 聽完正反方論述，純粹基於邏輯和事實裁決
- 逐點評估正反方的論據強度
- 給出 0-100 的分數，嚴格按照以下權重：
  - 市場需求真實性（20%）
  - 商業模式可行性（20%）
  - 執行可行性（20%）
  - 競爭護城河（15%）
  - 財務合理性（15%）
  - 回應批評的品質（10%）
- 最後必須拆解此提案存活的核心前提（Critical Assumptions），按維度分類：市場、技術、財務、法規、競爭、團隊

你不帶感情。數字說了算。
語氣：法官式、「本庭認為...」、「證據不足以支持...」。
```

#### 判官 - 悲觀型

```
你是一個極度挑剔的頂級 VC 合夥人，管理過百億基金，見過太多失敗。

你的工作：
- 聽完正反方論述，用最嚴格的標準裁決
- 你預設立場是「不投」，需要被說服才會改變
- 評分偏低，70 分在你這裡等於「勉強及格」
- 特別關注：單位經濟模型、CAC/LTV、競爭壁壘
- 最後必須拆解此提案存活的核心前提（Critical Assumptions），按維度分類：市場、技術、財務、法規、競爭、團隊

你的口頭禪：「So what?」「Why now?」「Why you?」
語氣：冷酷、簡短、不浪費時間在客套上。
```

### 抽籤機制

每場辯論開始前：
1. 隨機抽 LLM 模型分配三個座位（Kimi / Claude / Minimax）
2. 隨機抽每個座位的性格（樂觀 / 普通 / 悲觀）
3. 記錄本場配置，確保報告可追溯

```json
{
  "session": 1,
  "config": {
    "judge": { "model": "kimi", "personality": "pessimist" },
    "pro": { "model": "claude", "personality": "optimist" },
    "con": { "model": "minimax", "personality": "normal" }
  }
}
```

## 前端設計

- **風格：** 白色底、乾淨、大留白、專業商務感
- **參考：** McKinsey 報告 / Notion / Linear 風格
- **字體：** Inter（sans-serif）
- **主色：** McKinsey Blue `#0066FF` + 深灰 `#333333`
- **背景：** 純白 `#FFFFFF` + 淺灰區塊 `#F8F9FA`
- **3D 插圖：** 辦公室 / 會議室場景點綴
- **不要：** sci-fi、暗色、花俏動畫

### 分析中頁面（Labor Illusion）

等待過程是產品體驗的核心。用戶要看到 AI 真的在開會：

- **Phase 1 辯論**：實時 Stream 文字流，終端機風格（深色區塊嵌在白底頁面中）
  - 左側顯示當前座位排列（誰是判官 / 正方 / 反方）
  - 中間即時串流辯論文字（打字機效果）
  - 右側顯示回合進度（Round 1/5、場次 1/3）
- **Phase 2 壓測**：進度條 + 數字跳動
  - 「正在測試假設 3/10：市場規模 > 1000 萬用戶」
  - 即時跳動的 PASS/FAIL 計數器
  - 小動畫：綠色 ✓ 和紅色 ✗ 不斷出現
- **Phase 3 生成**：雷達圖從 0 展開的動畫 + 分數翻牌

### 圖表

- **雷達圖：** Chart.js，線條纖細，`#0066FF` 填充 + `#333333` 邊框
- **六個維度：** 市場、技術、財務、法規、競爭、團隊

### 頁面

| 頁面 | 內容 |
|------|------|
| 首頁 | 產品介紹 + 輸入提案框 |
| 分析中 | 實時辯論串流 + 壓測進度 + 報告生成動畫 |
| 報告頁 | 分數 + 雷達圖 + 模糊遮罩區（付費解鎖） |
| PDF | server 端生成，付費後可下載 |

## 技術棧

- **後端：** Node.js（零依賴風格，跟 claw-war 一致）
- **前端：** 單頁 HTML + CSS + JS（不用框架）
- **LLM API：** Claude / Kimi / Minimax 直接 HTTP 呼叫
- **圖表：** Chart.js（CDN 引入）
- **支付：** x402 協議（Base 鏈 USDC）
- **PDF 生成：** server 端 HTML → PDF
- **部署：** EC2
